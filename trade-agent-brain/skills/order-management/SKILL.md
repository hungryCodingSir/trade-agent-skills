# 订单管理技能

## 概述

本技能处理跨境电商平台的所有订单相关查询。
支持三种用户角色，每种角色有不同的数据访问权限。

## 使用指引

### 1. 判断查询类型

根据用户意图确定查询类型：
- **订单状态**：查询某个订单的当前进度（使用 `query_order_status` 工具）
- **订单明细**：商品列表、金额、收货地址
- **支付状态**：付款确认、退款进度
- **订单历史**：过往订单汇总

### 2. 角色差异化响应策略

**买家视角：**
- 展示订单状态、物流追踪、支付信息
- 隐藏供应商成本和利润信息
- 重点突出预计送达日期
- 延迟超过 7 天时主动建议联系客服

**卖家视角：**
- 展示履约状态和待发货订单
- 包含买家联系信息以便沟通
- 高亮需要立即处理的订单（已付款未发货）
- 展示收入和佣金明细

**管理员视角：**
- 完整数据访问，包括买卖双方信息
- 展示审计轨迹和状态变更历史
- 支持订单修改和状态覆盖操作

### 3. 工具使用

使用 `query_order_status` 工具，传入订单号（格式：`ORD20240101001`）。

如果用户未提供订单号：
- 请求用户提供完整订单号
- 建议从购买记录或订单确认邮件中查找

### 4. 响应格式

按以下结构组织回复：
1. **状态摘要** — 一句话描述当前状态
2. **关键信息** — 商品、金额、日期
3. **后续步骤** — 用户接下来可以期待什么或需要做什么

### 5. 异常处理

- 订单未找到 → 请用户核实订单号是否正确
- 服务超时 → 致歉并建议稍后重试
- 权限不足 → 解释角色权限限制

# 订单管理技能

## 概述

本技能处理跨境电商平台的所有订单相关查询。
支持三种用户角色，每种角色有不同的数据访问权限。

## 适用范围

本技能指导主 Agent 处理**简单订单任务**：
- ✅ 查询单个订单的状态、支付情况、商品明细
- ✅ 引导用户提供订单号
- ✅ 根据角色差异化展示单个订单信息
- ✅ 处理单个订单的异常情况（未找到、超时等）

以下场景请委派给 `order-specialist` 子智能体：
- ❌ 多个订单的交叉对比（如"对比我最近3个订单"）
- ❌ 订单历史趋势分析（如"我上个月总共花了多少"）
- ❌ 批量订单数据汇总和报表生成

## 边界判断

- "查一下 ORD001 和 ORD002 的状态" → 分别查询即可，**主Agent自行处理**
- "对比 ORD001 和 ORD002 哪个更划算" → 需要交叉分析，**委派 order-specialist**
- "我有一个订单好像出问题了" → 单订单排查，**主Agent自行处理**

## 使用工具

本技能使用 `query_order_status` 工具，传入订单编号（格式：`ORD20240101001`）即可获取完整订单数据。

## 使用指引

### 1. 判断查询类型

根据用户意图确定查询类型：
- **订单状态**：查询某个订单的当前进度（使用 `query_order_status` 工具）
- **订单明细**：商品列表、金额、收货地址
- **支付状态**：付款确认、退款进度
- **订单历史**：过往订单汇总

### 2. 角色差异化响应策略

**买家视角：**
- 展示订单状态、物流追踪、支付信息
- 隐藏供应商成本和利润信息
- 重点突出预计送达日期
- 延迟超过 7 天时主动建议联系客服

**卖家视角：**
- 展示履约状态和待发货订单
- 包含买家联系信息以便沟通
- 高亮需要立即处理的订单（已付款未发货）
- 展示收入和佣金明细

**管理员视角：**
- 完整数据访问，包括买卖双方信息
- 展示审计轨迹和状态变更历史
- 支持订单修改和状态覆盖操作

### 3. 响应格式

按以下结构组织回复：
1. **状态摘要** — 一句话描述当前状态
2. **关键信息** — 商品、金额、日期
3. **后续步骤** — 用户接下来可以期待什么或需要做什么

### 4. 异常处理

- 订单未找到 → 请用户核实订单号是否正确
- 服务超时 → 致歉并建议稍后重试
- 权限不足 → 解释角色权限限制
